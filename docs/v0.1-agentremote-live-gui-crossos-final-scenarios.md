Automated evidence is strong. The remaining work is live user-facing evidence.
The key rule: do not count a stale browser tab. Every GUI screenshot must come
from a freshly started master/controller whose backend API is reachable.

## Gate L0: Live GUI Session Guard

Before any screenshot or interaction:

1. Start a fresh local slave/share and master/controller GUI.
2. Record the exact commands and ports.
3. Verify these endpoints from the host:

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:<ui-port>/api/bootstrap
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:<ui-port>/api/dashboard
```

Pass criteria:

- both endpoints return HTTP 200;
- browser URL matches the live UI port;
- the page is reloaded after the server is started;
- the report includes the API responses with secrets redacted.

Fail criteria:

- screenshot is captured from a tab whose backend is not listening;
- Dashboard/Inbox shows an offline message and is counted as success.

## Gate L1: Core GUI File Workflows

Required workflows:

- upload a file to the currently open remote folder without selecting a remote
  file;
- upload a folder;
- download a file;
- download a folder;
- create a remote folder;
- rename and move a non-sensitive test item;
- delete only a disposable test item after explicit local test setup;
- repeat at least one transfer with Korean and accented filenames.

Required evidence:

- two-pane screenshot before transfer;
- upload plan screenshot;
- upload completion screenshot;
- download completion screenshot;
- folder-create screenshot;
- rename/move screenshot;
- hash comparison for one upload and one download.

Pass criteria:

- destination behavior is obvious;
- buttons remain visible;
- no unintended popup/window opens;
- free-space display remains visible and updates.

## Gate L2: Queue, Cancel, Speed, ETA

Required workflows:

- start upload and download concurrently;
- queue at least three entries;
- cancel one selected queued or running job;
- prove other jobs continue or complete;
- capture speed and ETA while active.

Required evidence:

- active transfer monitor screenshot;
- queued stack screenshot;
- canceled job state screenshot/API JSON;
- hash comparison for a non-canceled job;
- transfer session/log paths.

Pass criteria:

- cancel only affects the selected job;
- canceled jobs are structured as canceled;
- active jobs show speed and ETA.

## Gate L3: Dashboard, Inbox, Process Control

Required workflows:

- dashboard overview;
- node detail;
- process list;
- route/call/approval summaries;
- inbox view;
- stop one disposable project-scoped process and prove another remains alive.

Required evidence:

- live `/api/dashboard` JSON with secrets redacted;
- dashboard screenshot;
- inbox screenshot;
- process table before and after stop;
- no orphan process after cleanup.

Pass criteria:

- dashboard quickly answers what is connected, what is running, and what last
  moved;
- process stop is scoped to the selected process;
- stale/offline GUI pages show a clear restart/reload message.

## Gate X1: Native Cross-OS Filename Validation

Run when hosts are available. If not, mark the exact missing host pair as
`BLOCKED`.

Required pairs:

- Windows to Linux or Linux to Windows;
- macOS to Windows/Linux or Windows/Linux to macOS.

Datasets:

- Korean filenames;
- Japanese/Chinese filenames;
- NFC/NFD accent variants;
- case-collision candidates;
- long names near OS limits.

Required evidence:

- OS/filesystem inventory;
- source and destination listings;
- manifest and SHA-256 hashes;
- collision/normalization warning text.

Pass criteria:

- no silent overwrite;
- filenames remain readable;
- normalization collisions produce clear errors.
