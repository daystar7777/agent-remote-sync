# v0.1 Validation Scenarios

This document is the release-candidate validation plan for
agent-remote-sync / `agentremote`. It complements `docs/usage-scenarios.md`, which
maps many behaviors to automated tests. This plan focuses on end-to-end usage,
manual GUI checks, fault injection, and release confidence.

For the larger multi-environment release gate, use
[`docs/v0.1-fullscale-test-scenarios.md`](v0.1-fullscale-test-scenarios.md).

## Roles

- Scenario author: Codex.
- Test executor: DeepSeek.
- Fix owner after test report: Codex.

DeepSeek should execute these scenarios, record results, and avoid broad code
changes unless a tiny diagnostic patch is unavoidable. Codex will review the
report and make fixes.

## Result Report Format

Write results to:

```text
AIMemory/test-results_YYYYMMDD-HHMMSS-v01-scenarios-deepseek.md
```

Use this shape for each scenario:

```markdown
## Scenario A1: Fresh Setup

- Status: PASS / FAIL / BLOCKED / NOT_RUN
- Environment: Windows/macOS/Linux, Python version, shell, browser
- Commands:
  - `...`
- Expected:
  - ...
- Actual:
  - ...
- Evidence:
  - paths, screenshots, log snippets, dashboard URL, relevant JSON
- Issue severity: P0/P1/P2/P3 or none
- Minimal reproduction:
  - ...
```

Severity guide:

- P0: data loss, path escape, secret leak, unsafe remote execution, release
  blocker.
- P1: common user flow broken, transfer/handoff unreliable, install broken.
- P2: confusing UX, missing status, recoverable edge-case failure.
- P3: wording, polish, low-risk docs issue.

## Baseline Commands

Run these first from the repo root:

```powershell
python -m compileall -q src tests deepseek-test smoke.py
python -m pytest tests deepseek-test -q
python smoke.py
python -m unittest discover -s tests
git diff --check
```

Fresh package install smoke:

```powershell
$tmp = Join-Path $env:TEMP ('ars-validation-' + [guid]::NewGuid().ToString('N'))
$wheelDir = Join-Path $tmp 'wheel'
$venv = Join-Path $tmp 'venv'
$proj = Join-Path $tmp 'project'
$cfg = Join-Path $tmp 'cfg'
New-Item -ItemType Directory -Force $wheelDir, $proj | Out-Null
python -m pip wheel . -w $wheelDir --no-deps
python -m venv $venv
$py = Join-Path $venv 'Scripts\python.exe'
$agent = Join-Path $venv 'Scripts\agentremote.exe'
$wheel = (Get-ChildItem $wheelDir -Filter *.whl | Select-Object -First 1).FullName
& $py -m pip install $wheel
& $agent --help
& $agent doctor
& $agent install-work-mem --root $proj
$env:AGENTREMOTE_HOME = $cfg
& $agent daemon profile save --root $proj --name demo --host 127.0.0.1 --port 17171
& $agent daemon profile list --root $proj
& $agent daemon profile remove --root $proj --name demo
```

## Suggested Execution Order

1. Run Baseline Commands.
2. Run A/B/H scenarios to ensure local process lifecycle is stable.
3. Run C scenarios in the browser before moving to headless-only tests.
4. Run D/E/F/K project sync, handoff, approvals, topology scenarios.
5. Run G/I/J security and failure scenarios last because they may intentionally
   block clients, leave partials, or stop processes.
6. Finish with L docs/release review.

## Manual Scenario Details

### A1/A2: Fresh Setup and Safe Uninstall

Use a temp root. Confirm `setup`/`bootstrap` can install AIMemory, and
`uninstall` only prints/removes explicitly requested local state.

```powershell
$root = Join-Path $env:TEMP ('ars-a-' + [guid]::NewGuid().ToString('N'))
New-Item -ItemType Directory -Force $root | Out-Null
python -m agentremote setup --root $root --install yes --no-network-check
python -m agentremote doctor
python -m agentremote status --root $root
python -m agentremote uninstall --root $root --project-state
```

Expected:

- `AIMemory` exists after setup.
- Status shows AIMemory installed.
- Uninstall does not delete AIMemory unless `--purge-memory` is explicit.

### B/C: Local Two-Host Playground

Use two temp roots and one config home:

```powershell
$base = Join-Path $env:TEMP ('ars-play-' + [guid]::NewGuid().ToString('N'))
$local = Join-Path $base 'local'
$remote = Join-Path $base 'remote'
$cfg = Join-Path $base 'cfg'
New-Item -ItemType Directory -Force $local,$remote,$cfg | Out-Null
$env:AGENTREMOTE_HOME = $cfg
python -m agentremote install-work-mem --root $local
python -m agentremote install-work-mem --root $remote
```

Terminal 1:

```powershell
python -m agentremote share --root $remote --host 127.0.0.1 --port 17171 --password test --firewall no --no-console
```

Terminal 2:

```powershell
python -m agentremote connect lab 127.0.0.1 17171 --password test
python -m agentremote open lab --local $local
```

Expected:

- Browser opens automatically.
- Upload/download works even when only a destination folder is displayed and no
  row is selected.
- Long file lists do not push action buttons below reach.
- Transfer monitor shows queued/running/done/error/cancel plus speed and ETA.

### D: Project Sync

Create files under `$local`, including ignored dirs and conflicts under
`$remote`.

```powershell
New-Item -ItemType Directory -Force "$local\.git","$local\node_modules","$local\src" | Out-Null
'keep' | Set-Content "$local\src\app.txt"
'skip' | Set-Content "$local\node_modules\skip.txt"
'remote-old' | Set-Content "$remote\src\app.txt"
python -m agentremote sync-project lab --local $local /project
python -m agentremote sync-project lab --local $local /project --yes
python -m agentremote sync-project lab --local $local /project --yes --overwrite
```

Expected:

- First command plans only.
- Conflict blocks until `--overwrite`.
- Ignored directories are listed or excluded as expected.

### E/F: Handoff, Worker, Approval

```powershell
python -m agentremote tell lab "Inspect /project and report back" --path /project
python -m agentremote handoff lab "$local\src\app.txt" "Review this file and report back"
python -m agentremote approvals policy --root $remote --mode ask
python -m agentremote worker-policy init --root $remote
python -m agentremote worker-policy allow python-version python --args-pattern "*--version*" --root $remote
python -m agentremote ask lab "agentremote-run: python --version" --auto-run --wait-report --timeout 30
python -m agentremote worker --root $remote --once --execute yes
```

Expected:

- Handoff records exist locally and remotely.
- Approval mode creates requests when required.
- Worker executes only allowlisted `agentremote-run:` commands.
- Reports are matched to call records when callback/parent IDs are present.

### G: Security Checks

Do not perform a real DDoS. Use small deterministic probes:

```powershell
python -m agentremote connect bad 127.0.0.1 17171 --password wrong
python -m agentremote connect readonly 127.0.0.1 17171 --password test --scopes read,handoff
python -m agentremote send readonly "$local\src\app.txt" /blocked
python -m agentremote policy allow-tailscale
python -m agentremote policy list
```

Expected:

- Wrong password fails without storing a connection.
- Scoped token cannot write.
- Tailscale whitelist helper stores CIDR policy only, no secrets.
- Dashboard payload contains no token/password material.

### I/J: Fault Injection

Use small files for permission/storage tests unless the platform makes a larger
file cheap. Record exact OS behavior because Windows/macOS/Linux differ.

Recommended checks:

- Create NFC and NFD variants using Python `unicodedata`, then push/pull.
- Make a target directory read-only and attempt upload.
- Start a transfer, cancel it, confirm partial file remains and cleanup removes
  only stale partials.
- Resume a partial transfer and verify completed content hash.

### K: Topology and Routes

```powershell
python -m agentremote map --root $local
python -m agentremote status --root $local
python -m agentremote nodes list
python -m agentremote topology show
python -m agentremote route set lab 127.0.0.1 17171 --priority 5
python -m agentremote route probe lab
python -m agentremote route list
```

Expected:

- User can infer current project, known nodes, best route, recent calls,
  approvals, process counts, and daemon profiles.
- Priority controls selection before latency.
