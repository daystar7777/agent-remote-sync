# v0.1 Full-Scale Lab Runbook

This runbook turns `docs/v0.1-fullscale-test-scenarios.md` into an executable
multi-machine validation plan. It is meant for the next DeepSeek phase after the
Windows-only full-scale pass.

For the repeatable Linux-container version of this validation, see
[`docs/docker-fullscale-lab.md`](docker-fullscale-lab.md).
For the final no-shortcuts release gate across automated, Docker, VM, real
host, GUI, security, soak, daemon, and packaging layers, see
[`docs/v0.1-true-fullscale-test-scenarios.md`](v0.1-true-fullscale-test-scenarios.md).

## Output

Write the final report to:

```text
AIMemory/test-results_YYYYMMDD-HHMMSS-fullscale-lab-deepseek.md
```

Include:

- Machine inventory and OS versions.
- Network shape: localhost, LAN, Tailscale, or public HTTPS.
- Commands run on each machine.
- PASS/FAIL/BLOCKED per scenario.
- Screenshots for GUI/dashboard scenarios.
- Hashes for large-file transfer scenarios.
- Minimal repro for every failure.

## Machine Matrix

Use at least two machines for RC lab validation and three for stable validation.

| Node | Preferred OS | Role |
|------|--------------|------|
| A | Windows 10/11 | Controller, GUI, local project |
| B | macOS | Swarm daemon, Unicode/NFD filename target |
| C | Linux | Swarm daemon, many-file/large-file soak target |

Minimum acceptable RC fallback:

- A and B only, or A and C only.
- Mark the missing OS pairings as `BLOCKED: environment unavailable`.

Network modes:

1. Localhost for single-machine sanity.
2. LAN or VM host-only route.
3. Tailscale route if available.
4. HTTPS/TLS pinned fingerprint route for one pair.

Do not expose a daemon to the public internet without firewall/VPN/reverse proxy
controls.

## Lab Data Generator

Run from a disposable test root. Adjust sizes if the disk is small.
Do not intentionally fill a real disk.

PowerShell:

```powershell
$root = Join-Path $env:TEMP ('ars-lab-data-' + [guid]::NewGuid().ToString('N'))
New-Item -ItemType Directory -Force $root | Out-Null
python tools\generate_fullscale_lab_data.py --root $root --many-count 5000 --large-size-mib 1024
```

bash/zsh:

```bash
root="${TMPDIR:-/tmp}/ars-lab-data-$(date +%s)"
python tools/generate_fullscale_lab_data.py --root "$root" --many-count 5000 --large-size-mib 1024
```

Use `--large-size-mib 0` to skip the large-file dataset on small disks.
Use a smaller value such as `--large-size-mib 64` for a quick rehearsal before
the 1 GiB resume run.

## Ring L0: Per-Machine Setup

Run on every node:

```powershell
python --version
python -m pip install -e .
python -m agentremote doctor
python -m agentremote setup --root . --install yes --no-network-check
python -m agentremote status --root .
```

Expected:

- Python 3.10+.
- `agentremote` CLI works.
- AIMemory is installed.
- `status` is understandable and does not crash.

## Ring L1: Pairwise Connectivity

For each pair A->B, B->A, A->C, C->A, and B<->C when available:

On receiver:

```powershell
python -m agentremote share --root <receiver-root> --host 0.0.0.0 --port 7171 --password <lab-password> --firewall no
```

On sender:

```powershell
python -m agentremote connect <alias> <receiver-ip> 7171 --password <lab-password>
python -m agentremote map --root .
python -m agentremote status --root .
python -m agentremote send <alias> <DS-small> /incoming/small --overwrite
python -m agentremote pull <alias> /incoming/small ./received-small --overwrite
```

Expected:

- Alias normalized as `::<alias>`.
- Push/pull hashes match.
- Map/status show the node and route clearly.
- Password is not stored in connection/profile files.

## Ring L2: Cross-OS Filename Validation

Run each direction available:

- Windows -> macOS
- macOS -> Windows
- Windows -> Linux
- Linux -> Windows
- macOS -> Linux
- Linux -> macOS

Commands:

```powershell
python -m agentremote push <alias> <DS-unicode> /unicode --overwrite
python -m agentremote pull <alias> /unicode ./received-unicode --overwrite
python tools\inspect_unicode_names.py .\received-unicode
```

Use `python tools/inspect_unicode_names.py ./received-unicode` on bash/zsh.

Expected:

- No mojibake.
- NFC/NFD variants do not silently overwrite each other.
- Any case-insensitive collision is reported as a conflict.
- Console output remains readable or safely replaces unencodable characters.

## Ring L3: GUI and Dashboard On A

On A:

```powershell
python -m agentremote open <alias> --local <controller-root>
```

Manual checks:

- Remote left pane and local right pane are obvious.
- Free space is visible for both sides.
- Upload file/folder from DS-small and DS-unicode.
- Download remote file/folder.
- New folder, rename, move, delete, and root delete rejection.
- Long folder from DS-many keeps Upload/Download controls visible.
- Transfer monitor shows queued/current/done/error/cancel plus speed and ETA or
  remaining time.
- Dashboard shows nodes, calls, approvals, processes, daemon profiles, worker
  policy, and recent activity.
- Dashboard payload and DOM do not expose passwords, bearer tokens, token hashes,
  or command fingerprints.

Attach screenshots to the result report when possible.

## Ring L4: Handoff, Worker, Approval

On receiver:

```powershell
python -m agentremote worker-policy init --root .
python -m agentremote worker-policy allow python-version python --args-pattern "*--version*" --timeout 30 --root .
python -m agentremote approvals policy --root . --mode ask
```

On controller:

```powershell
python -m agentremote tell <alias> "Inspect /incoming/small and report back" --path /incoming/small
python -m agentremote handoff <alias> <DS-small> "Review this folder and report back"
python -m agentremote ask <alias> "agentremote-run: python --version" --auto-run --wait-report --timeout 120
```

On receiver:

```powershell
python -m agentremote approvals list --root .
python -m agentremote worker --root . --once --execute yes
python -m agentremote inbox --root .
```

Expected:

- Approval requests distinguish local/remote origin.
- Approved work unblocks.
- Non-allowlisted commands are blocked.
- Reports return and update call records.

## Ring L5: Security and Failure

Use deterministic, small probes only.

```powershell
python -m agentremote connect bad <receiver-ip> 7171 --password wrong
python -m agentremote connect readonly <receiver-ip> 7171 --password <lab-password> --scopes read,handoff
python -m agentremote send readonly <DS-small> /blocked
python -m agentremote policy allow-tailscale
python -m agentremote policy list
```

Also verify:

- Traversal paths are rejected.
- Reserved `.agentremote*` dirs are hidden/protected.
- TLS self-signed fingerprint works with the correct fingerprint and fails with
  the wrong fingerprint.
- Bad login flood returns blocked/429 without real DDoS behavior.
- Permission-denied write returns a structured error and failed session.
- Insufficient-storage preflight uses mocked/free-space checks or a disposable
  test volume only.

## Ring L6: Large-File Resume

Run only on a disposable network/storage path.

Upload:

```powershell
python -m agentremote push <alias> <DS-large>\large-1gib.bin /large --overwrite
```

Interruption test:

1. Start upload/download.
2. Stop the sender or receiver process mid-transfer.
3. Restart `share`.
4. Run the same command again.
5. Compare SHA-256 with `large-1gib.sha256`.

Expected:

- Partial file remains until resumed or cleaned.
- Resume continues rather than starting from zero when possible.
- Final hash matches.
- Session/log files record interruption and completion.

## Ring L7: Many-File Soak

```powershell
python -m agentremote sync-project <alias> --local <DS-many> /many
python -m agentremote sync-project <alias> --local <DS-many> /many --yes
python -m agentremote sync-project <alias> --local <DS-many> /many --yes --compare-hash
python -m agentremote cleanup --root .
```

Expected:

- 5,000+ files complete without unbounded memory growth.
- Repeat sync is mostly skipped or hash-compared.
- Logs rotate.
- AIMemory contains summaries/pointers, not per-file log spam.

## Ring L8: Daemon and Route Persistence

```powershell
python -m agentremote daemon profile save --root . --name lab-node --host 0.0.0.0 --port 7171
python -m agentremote daemon profile list --root .
python -m agentremote daemon status --root .
python -m agentremote route set <alias> <receiver-ip> 7171 --priority 10
python -m agentremote route probe <alias>
python -m agentremote route list
python -m agentremote topology show
```

Expected:

- Profiles store no password/token.
- Status correlates running/not-running processes.
- Route priority wins before latency.
- Offline/stale state is understandable after stopping a daemon.

## Exit Criteria

For v0.1 stable:

- No P0/P1 findings.
- Any P2 has a fix or a clearly documented limitation.
- R7/R8 from the high-level scenario doc are no longer simply `BLOCKED` unless
  the missing machine/disk environment is explicitly unavailable.
- Cross-OS filename behavior has at least one Windows<->macOS or
  Windows<->Linux real pairing result.
- Large-file resume has at least one successful interrupted/resumed transfer.
- No secret exposure appears in configs, logs, AIMemory, dashboard payloads, or
  browser DOM.
