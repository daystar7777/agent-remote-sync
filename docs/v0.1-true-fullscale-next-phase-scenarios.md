# v0.1 True Full-Scale Next-Phase Scenarios

This addendum starts after the automated Docker integrity gate is passing. It
focuses on the remaining confidence gaps that Linux Docker cannot prove:
browser UX, long-running soak, cross-OS filesystems, service lifecycle, and
agent handoff call/return behavior.

## Current Automated Baseline

The runner now validates:

- Clean Docker volumes and persistent Docker volumes.
- Unicode transfer plus `unicode-manifest.json` hash verification.
- `sync-project --delete` mirror behavior with a deliberately stale remote file.
- Wrong password rejection and scoped-token write denial.
- Many-file sync and large-file upload rehearsals.

The next phase should not repeat only command-exit checks. Every scenario below
must include direct evidence: screenshot, hash, manifest, call record, process
record, or AIMemory file path.

## P1: Docker Soak Gate

Run:

```powershell
python tools\run_docker_fullscale.py --fresh --many-count 5000 --large-size-mib 1024 --down-volumes
```

If disk/time is insufficient, run:

```powershell
python tools\run_docker_fullscale.py --fresh --many-count 2000 --large-size-mib 256 --down-volumes
```

Pass criteria:

- Report shows `0 FAIL`.
- Unicode manifest hashes pass.
- Mirror stale-delete pass.
- Large uploaded file hash is verified against the `.sha256` file or the runner
  is extended to verify it.
- No `rate_limited`, `hash_mismatch`, `size_mismatch`, or unbounded retry loop.

## P2: Browser GUI Evidence Gate

Start a Docker node or local slave, then open the GUI:

```powershell
agentremote connect gui-node 127.0.0.1 7171 --password <password>
agentremote open gui-node --local <local-root>
```

Required screenshots:

- File browser with remote pane and local pane visible.
- Transfer monitor during active upload or download.
- Dashboard overview with nodes/processes/routes/calls.
- Detail panel for a node or call.
- Approval list or worker policy panel, with secrets redacted.

Required actions:

- Upload file with only remote folder context selected.
- Upload folder.
- Download file.
- Download folder.
- Create folder.
- Rename, move, delete.
- Cancel one queued/current transfer while another remains valid.
- Confirm transfer speed/ETA/progress is visible.
- Confirm local/remote free space is visible.

Pass criteria:

- Controls stay visible in long file lists.
- No password, bearer token, token hash, or command fingerprint appears in DOM,
  API JSON, screenshots, or logs.
- Cancel affects only the selected job.

## P3: Resume / Restart Fault Gate

Use at least a 256 MiB file locally, 1 GiB if practical.

Required cases:

- Kill sender during upload, restart, resume, final hash match.
- Kill receiver during upload, restart, retry/resume, final hash match.
- Kill sender during download, restart, resume, final hash match.
- Corrupt a partial manually, retry, final hash match and partial cleanup.
- Run concurrent upload and download while polling dashboard transfer state.

Pass criteria:

- Partial state is either resumed or discarded with a structured reason.
- No stale `.agentremote_partial` entry remains after a successful finish.
- Transfer logs rotate and stay bounded.

## P4: Cross-OS Filename Gate

Run at least two real or VM pairs:

- Windows -> Linux or Linux -> Windows.
- macOS -> Windows/Linux or Windows/Linux -> macOS.

Required datasets:

- Korean filenames.
- Japanese/Chinese filenames.
- Accent filenames with NFC and NFD variants.
- Long names near OS limits.
- Case collision candidates on case-insensitive filesystems.

Pass criteria:

- No silent overwrite after normalization.
- Collisions are reported clearly.
- Pulled filenames remain readable on the target OS.
- Manifest/hash evidence is included.

## P5: Agent Handoff / Call-Return Gate

Required cases:

- `tell` instruction only.
- `handoff` file plus task.
- `ask --wait-report` with worker callback.
- Worker allowed command.
- Worker blocked command.
- Worker timeout.
- Local approval request.
- Remote approval request.

Pass criteria:

- Both local and remote AIMemory record the external handoff context.
- Dashboard or CLI shows call states: sent, claimed/running, reported, failed, or
  timeout.
- Approval decision reaches the waiting execution path.
- Auto-run does not execute outside the worker allowlist.

## P6: Daemon / Reboot / Uninstall Gate

Required cases:

- Multiple project daemons on one host.
- Saved daemon profiles list/remove.
- Dashboard stop/forget for safe process entries.
- Service install dry-run for Windows/macOS/Linux.
- Reboot persistence if a real host is available.
- Uninstall dry-run and uninstall documentation review.

Pass criteria:

- Profiles store no password/token.
- Stale process detection works after forced stop.
- Uninstall preserves AIMemory by default and removes only explicitly selected
  agent-remote-sync state.

## Required Report

Write:

```text
AIMemory/test-results_YYYYMMDD-HHMMSS-true-fullscale-nextphase-deepseek.md
```

Include:

- Environment inventory.
- Exact commands.
- Docker report paths.
- Screenshot paths.
- File hashes.
- AIMemory handoff/call/report paths.
- PASS/FAIL/BLOCKED/NOT_RUN matrix.
- P0/P1 issue list.

If a scenario cannot run, mark it `BLOCKED` with the exact missing environment.
Do not count `BLOCKED` as `PASS`.
